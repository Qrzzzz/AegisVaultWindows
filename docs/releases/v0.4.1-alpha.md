# AegisVault v0.4.1-alpha

This is a release-consistency and packaging-verification patch over `v0.4.0-alpha`.

It remains an alpha release. Keep backups for any important data.

## What Changed

- Aligned package metadata, runtime version constants, documentation references and release artifact naming for `v0.4.1-alpha`.
- Restored normal source, Markdown, workflow and PowerShell formatting where needed.
- Repaired CI workflow formatting.
- Added source-health tests to detect unparsable or collapsed files.
- Added version-consistency tests for package metadata, runtime version and documentation.
- Added `scripts/verify_release.ps1` for repeatable local release checks.
- Updated QA and release checklists for the `0.4.1-alpha` line.

## Compatibility

- No change to the `AGV1.` text token format.
- No change to the `.agv` chunked file container format.
- No change to legacy recovery formats.
- Data created by `v0.4.0-alpha` should remain compatible with `v0.4.1-alpha`.

## Downloads

- `AegisVault-v0.4.1-alpha-win64.zip` - standalone Windows x64 build. Extract and run `AegisVault.exe`.

## Notes

- Alpha release: interface, packaging and compatibility behavior may still change.
- Do not use for irreplaceable data without verified backups.
- Requires Windows 10 x64 or later.
- See README for security design and usage.
